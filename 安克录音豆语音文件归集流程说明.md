# 安克录音豆语音文件自动归集与重命名流程说明

## 一、项目背景

目前安克录音豆一次只能绑定一个飞书账号，录音文件和智能纪要会自动生成在对应置业顾问个人飞书账号的个人文件夹中。

在实际业务中，每天会有多位置业顾问接待客户，并通过安克录音豆生成录音文件和智能纪要。由于这些文件分散在不同个人账号下，且生成文件本身没有统一的 token 或固定路径可以直接定位，后续统一归档、核对、命名和处理成本较高。

本程序的目标是：

通过企业飞书授权机制，获取每位置业顾问个人飞书文件夹中的录音文件和智能纪要，根据来访时间与接访信息表进行匹配，自动完成文件识别、重命名，并复制到指定统一文件夹中，方便后续集中处理和管理。

## 二、整体业务流程

### 1. 置业顾问加入企业飞书

每位置业顾问需要加入企业飞书组织，并使用企业飞书账号作为日常工作账号。

这样可以保证后续程序能够在企业授权范围内识别对应用户，并通过授权访问其个人飞书文件。

### 2. 置业顾问绑定自己的安克录音豆

每个安克录音豆绑定对应置业顾问的企业飞书账号。

绑定后，安克录音豆每天生成的内容会自动上传到该置业顾问个人飞书空间中，包括：

- 原始录音文件
- 安克录音豆自动生成的智能纪要
- 可能存在的相关转写或摘要文件

### 3. 系统发送授权邀请

管理员可在系统中按用户名搜索企业通讯录成员，并向指定置业顾问发送授权邀请。

置业顾问点击授权链接后，完成飞书账号授权。授权成功后，系统会保存该用户的授权信息，后续可在系统中选择该用户进行文件搜索和处理。

业务上可以理解为：

置业顾问同意系统代为检索其飞书个人空间中由安克录音豆生成的相关文件。

### 4. 系统定位每日生成文件

管理员在系统中选择已授权的置业顾问账号，并按日期、关键词或文件类型搜索其飞书文件。

系统会在该用户的飞书云文档或文件空间中查找当天生成的录音文件和智能纪要。

重点定位依据包括：

- 文件生成时间
- 文件更新时间
- 文件名称
- 文件类型
- 安克录音豆相关关键词
- 用户个人文件空间中的搜索结果

由于安克录音豆文件没有固定 token，系统不能直接按固定文件 ID 定位，而是通过用户授权后的飞书搜索能力，在用户个人文件夹中动态查找每日新增文件。

### 5. 导入或读取接访信息表

业务侧提供当天接访信息表，表中通常包含：

- 置业顾问姓名
- 客户姓名
- 来访时间
- 接访项目
- 客户手机号或其他识别信息
- 接访备注

系统会以“准确来访时间”为核心匹配依据，将接访信息表与录音文件生成时间进行比对。

### 6. 根据来访时间匹配录音文件

系统会将飞书中搜索到的录音文件、智能纪要，与接访信息表中的来访记录进行匹配。

匹配逻辑以业务可解释为主：

- 同一置业顾问名下的文件优先匹配该顾问的接访记录
- 文件生成时间接近客户来访时间
- 若存在多个候选文件，按时间距离优先选择
- 录音文件和智能纪要可作为同一接访记录的配套材料归集
- 匹配异常或无法确认的文件进入人工核对列表

### 7. 自动重命名文件

匹配成功后，系统根据接访信息表中的业务字段生成规范文件名。

示例命名规则：

```text
来访日期_来访时间_置业顾问_客户姓名_文件类型
```

例如：

```text
2026-05-07_14-30_张三_王先生_录音
2026-05-07_14-30_张三_王先生_智能纪要
```

这样业务人员无需打开文件，也能直接从文件名判断该文件对应哪一次接访。

### 8. 复制到指定统一文件夹

系统将匹配并重命名后的文件复制到指定飞书文件夹中。

该文件夹可作为每日接访录音和智能纪要的统一归档目录，便于后续：

- 质检复盘
- 客户跟进
- 销售过程管理
- 录音抽查
- 智能纪要集中处理
- 后续数据分析

系统采用“复制”而不是“移动”的方式，避免改变置业顾问个人飞书文件夹中的原始文件位置，降低对个人工作空间的影响。

## 三、角色分工

### 置业顾问

- 加入企业飞书
- 使用企业飞书账号绑定安克录音豆
- 接收并完成系统授权
- 正常使用安克录音豆接待客户

### 管理员 / 运营人员

- 搜索置业顾问并发送授权邀请
- 选择已授权用户进行文件搜索
- 提供或上传接访信息表
- 发起文件匹配、重命名和复制
- 核对异常文件

### 系统

- 维护置业顾问飞书授权状态
- 检索置业顾问个人飞书文件
- 识别录音文件和智能纪要
- 根据来访时间匹配接访信息
- 自动重命名并复制到统一文件夹

## 四、关键技术设计简述

### 1. 飞书 OAuth 授权

系统通过飞书 OAuth 获取置业顾问授权。授权后，系统可以在用户同意的范围内搜索该用户飞书空间中的文件。

这一步解决的是：

文件在个人账号下，系统必须获得该用户授权后才能访问。

### 2. 企业通讯录搜索

系统通过企业飞书通讯录能力，按用户名搜索置业顾问，并向其发送授权邀请。

这一步解决的是：

管理员不需要手动输入复杂的 open_id，只需要按姓名找到人并发送邀请。

### 3. 用户级文件搜索

由于安克录音豆生成的文件没有提前可知的 token，系统无法直接按文件 ID 查找，只能基于已授权用户进行飞书云文档/文件搜索。

这一步解决的是：

在用户个人文件夹中动态定位当天生成的录音和纪要文件。

### 4. 时间匹配

系统以“来访时间”为核心，将接访信息表与录音文件生成时间进行匹配。

这一步解决的是：

录音文件名可能不规范，但生成时间与客户来访时间具有业务关联。

### 5. 文件复制与重命名

系统将匹配后的文件复制到统一文件夹，并按接访信息生成业务可读文件名。

这一步解决的是：

文件分散、命名混乱、后续处理不方便的问题。

## 五、最终效果

上线后，业务侧可以形成如下工作方式：

1. 置业顾问正常接待客户并使用安克录音豆
2. 录音和智能纪要自动进入其个人飞书文件夹
3. 管理员在系统中选择置业顾问和日期
4. 系统自动搜索当天文件
5. 系统根据接访信息表匹配客户来访记录
6. 系统自动重命名并复制到统一文件夹
7. 运营或管理人员在统一目录中查看和处理全部录音材料

## 六、业务价值

- 减少人工查找个人飞书文件的时间
- 降低录音文件漏归档、错归档风险
- 统一录音和智能纪要命名规范
- 便于后续质检、复盘和客户跟进
- 保留原始文件位置，不影响置业顾问个人使用
- 支持多置业顾问、多账号、多日常批量处理

## 七、注意事项

- 每位置业顾问都需要完成一次飞书授权
- 授权失效后需要重新授权
- 安克录音豆需绑定企业飞书账号
- 文件匹配依赖来访时间准确性
- 若同一时间段内接访过于密集，可能需要人工确认
- 统一归档文件夹需要提前配置好访问权限
